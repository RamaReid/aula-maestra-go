## Annual Planning Pattern

The annual plan is not theme-only.

Each class resolves as:

1. focus
2. operation
3. minimum evidence

Examples from the accepted annual logic:

- problem and hypothesis
- controls and evidence
- decision and ethics

This means the app should not treat annual lessons as just `lesson number + title`.

## Agenda Pattern

The agenda should let the teacher see:

- class number
- lesson theme
- development state
- direct path to work on the lesson

In the demo, the agenda is not detached from planning. It is the visible operational layer of the yearly plan.

## Sequence Pattern

The accepted sequence logic is cumulative.

For the demo case, classes 15 to 17 work because each one leaves a concrete partial product:

- precision and explanation
- reasoning and fallacy correction
- report improvement with references

The sequence is not just thematic continuity. It is production continuity.

## Class Pattern

The developed Popper lesson in the demo shows the expected class shape:

- purpose
- contents
- base bibliography
- reading material for students
- timed development
- minimum evidence
- equivalent recovery
- resources
- plan B
- situated adaptations

This is a very good class-level canon for the app because it is concrete and teachable.

## Reading Material Pattern

The accepted reading material in the demo is:

- prose, not bullet summary
- aligned to the class purpose
- conceptually serious
- readable for school level
- tied to a named bibliography

The reading does not try to replace the class. It prepares and supports the class production.

## Situated Case Pattern

The accepted case bank includes real agrotechnical situations such as:

- sweets and gelification
- sausage acidity and maturation
- process water chlorination
- sheep welfare and withdrawal period
- privacy and digital traceability

These are not decorative examples. They are the bridge between curriculum and school reality.

## Product Implication

For this subject and context, the app should preserve these expectations:

- yearly planning must already imply outputs
- sequence generation must preserve continuity
- class generation must produce something teachable
- reading material must support the exact lesson and not drift into generic exposition

## Change Warning

Do not replace this golden reference casually.

If a future canon changes it, the replacement must be clearly better in:

- pedagogical coherence
- situational relevance
- traceability
- usability inside the real app flow
